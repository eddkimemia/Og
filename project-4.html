PROJECT: STATUSPULSE

$ cat tech_loadout.txt

THE MISSION

Developers needed a reliable way to monitor service uptime and notify stakeholders during outages without paying for expensive enterprise tools.

THE APPROACH

I implemented a high-concurrency monitoring engine using BullMQ. The system pings user endpoints at configurable intervals and auto-generates a public status page.

THE CODE

const pingWorker = new Worker('pings', async job => {
  const { url, id } = job.data;
  try {
    const start = Date.now();
    await axios.get(url, { timeout: 5000 });
    await saveLog(id, Date.now() - start, 'up');
  } catch (e) {
    await sendAlert(id, 'down');
  }
});

THE OBSTACLES

Managing thousands of concurrent pings without overwhelming the network. I implemented rate-limiting and distributed the load across multiple worker nodes.

THE RESULTS

Currently monitoring 200+ endpoints with 99.9% reliability and sending downtime alerts via SMS/Email within 15 seconds of failure detection.
