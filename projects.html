PROJECT: INVOICEKENYA (ISSUE #01)

The Mission: Kenyan freelancers struggled with manual invoice tracking and high transaction fees. I built a tool to automate billing and integrate M-Pesa payments directly into the workflow.

The Tech Stack: React, Node.js, Express, PostgreSQL, Daraja API, PDFKit, Tailwind CSS.

// Daraja STK Push Integration
async function initiateSTKPush(phone, amount) {
  const response = await axios.post(DARAJA_URL, {
    BusinessShortCode: "174379",
    Amount: amount,
    PhoneNumber: phone,
    CallBackURL: "https://api.invoicekenya.com/mpesa-callback"
  });
  return response.data;
}

The Obstacles: Handling Daraja API callbacks reliably during peak network congestion was a major hurdle. I solved this by implementing a retry mechanism and a background worker to verify transaction status if the callback failed.

The Results: Reduced manual reconciliation by 90% for active users and successfully processed over 5,000 invoices in the first 3 months.

PROJECT: FORMCRAFT (ISSUE #03)

The Mission: Local businesses needed a low-cost alternative to Typeform that could be easily embedded in their existing sites without technical expertise.

The Tech Stack: React, Node.js, Express, MongoDB, AWS S3, Vanilla JS (embed runtime).

// Lightweight Embed Script
(function() {
  const script = document.currentScript;
  const formId = script.getAttribute('data-form-id');
  const container = document.createElement('div');
  container.id = `formcraft-${formId}`;
  script.parentNode.insertBefore(container, script);
  loadForm(formId, container);
})();

The Obstacles: Ensuring the embed script didn't conflict with host site CSS was difficult. I utilized Shadow DOM to isolate the form styles completely from the surrounding page.

The Results: Shipped 2 weeks ahead of schedule. Currently used by 15+ local startups for lead generation.

PROJECT: STATUSPULSE (ISSUE #04)

The Mission: I wanted to provide a reliable uptime monitoring tool for local developers who needed downtime alerts via both email and SMS.

The Tech Stack: Node.js, PostgreSQL, BullMQ (job queue), Nodemailer, Twilio, React, Chart.js.

// Background job for pinging endpoints
const pingWorker = new Worker('pings', async job => {
  const { url, id } = job.data;
  const start = Date.now();
  try {
    await axios.get(url, { timeout: 5000 });
    await saveLog(id, Date.now() - start, 'up');
  } catch (e) {
    await saveLog(id, 0, 'down');
    await sendAlert(id);
  }
});

The Obstacles: Managing a large number of concurrent pings without overwhelming the server. I used BullMQ with Redis to queue pings and process them with a pool of workers.

The Results: Monitoring 200+ endpoints with 99.9% reliability. Incident history view helped users identify recurring network patterns.
