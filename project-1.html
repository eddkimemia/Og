PROJECT: INVOICEKENYA

$ cat tech_loadout.txt
┌────────────┬────────────────────────────────────┐ │ Frontend │ React, Tailwind CSS │ │ Backend │ Node.js, Express │ │ Database │ PostgreSQL │ │ Infra │ Daraja API, PDFKit │ └────────────┴────────────────────────────────────┘

THE MISSION

Kenyan freelancers and small businesses struggled with manual invoice generation and slow payment confirmation. InvoiceKenya automates this process with native M-Pesa integration.

THE APPROACH

I built a PDF generation engine using PDFKit and integrated the Daraja API for STK Push callbacks. This allows clients to pay directly from the invoice link, with the system automatically marking it as paid upon successful callback.

THE CODE

async function initiateSTKPush(phone, amount) {
  const response = await axios.post(DARAJA_URL, {
    BusinessShortCode: "174379",
    Amount: amount,
    PhoneNumber: phone,
    CallBackURL: "https://api.invoicekenya.com/mpesa-callback"
  });
  return response.data;
}

THE OBSTACLES

Handling Daraja API timeouts and inconsistent callback delivery. I implemented a background poller using BullMQ to verify transaction status if the webhook wasn't received within 60 seconds.

THE RESULTS

Reduced manual reconciliation by 90% and processed over 5,000 invoices in the first 3 months with 99.8% payment reliability.
