How I Built This Comic Terminal Portfolio (And Why You Should Break UI Rules)

In a world of generic, minimalist developer portfolios, I wanted something that screamed personality. I chose to blend the gritty, utilitarian vibe of a Linux terminal with the vibrant, tactile aesthetic of a Silver Age comic book.

Rejecting Generic Portfolios

Most portfolios today look like a Vercel template. They're clean, they're fast, but they're forgettable. By breaking UI rules—using thick borders, off-kilter rotations, and high-contrast halftone patterns—I created a space that forces engagement.

The Technical Challenges

The biggest challenge was implementing the halftone dot pattern without using heavy images. I used pure CSS radial gradients to achieve the look.

.panel.halftone::after {
  content: "";
  position: absolute;
  inset: 0;
  background-image: radial-gradient(var(--black) 1px, transparent 1px);
  background-size: 4px 4px;
  opacity: 0.05;
}

Keeping the layout accessible while using `transform: rotate()` on panels required careful consideration. I ensured that the rotation was subtle enough (0.5deg) not to interfere with readability while still providing that "hand-laid comic" feel.

Performance & Accessibility

Despite the "chaotic" visual language, the site is incredibly lightweight. No frameworks, no heavy libraries—just vanilla HTML, CSS, and JS. Every panel is semantic, and the contrast ratios meet WCAG AA standards by using a dark-on-cream palette instead of pure white.

"The goal was to make it feel like an engineer who codes, but also an engineer who has a story to tell."

Lessons for Devs

Don't be afraid to build something weird. Technical competence is expected; personality is what gets you remembered.
