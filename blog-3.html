Switching from 'any' to 'unknown'

  • 1. Introduction
  • 2. The Deep Dive

  • 4. Lessons Learned

INTRODUCTION

I used to think 'any' was a harmless shortcut. Then a typo in a nested object property bypassed our type checks and caused a runtime crash for 500 users. That was the day I swore off 'any' forever.

THE DEEP DIVE

TypeScript's 'unknown' type is the safe alternative. It forces you to perform type checking before you can interact with the value. It's more work upfront, but it guarantees that your assumptions about external data are actually verified at runtime.

'any' is just a debt that you eventually have to pay back with interest in production.

CODE IMPLEMENTATION

function handleResponse(data: unknown) {
  if (isUserResponse(data)) {
    return data.userName;
  }
  throw new Error('Invalid data');
}

LESSONS LEARNED

Strict mode is your best friend. It caught three more hidden bugs during our refactor that would have eventually caused issues.
